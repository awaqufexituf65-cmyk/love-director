# 动态剧本生成方法论

## 核心原则

决策树不是套模板——是从用户数据中生长出来的。

```
用户数据 → 关键变量提取 → 分支构建 → 概率赋值 → 结局生成
   ↑                                                    ↓
   └──────────── 每个步骤的数据依据回溯 ─────────────────┘
```

---

## Step 1: 关键变量提取

### 筛选标准

| 标准 | 说明 | 阈值 |
|------|------|------|
| 自述-行为差距 | 用户声称的 vs WeFlow 数据揭示的差异程度 | 差距 ≥ 1 个标准差 |
| 跨关系一致性 | 变量在 ≥2 段关系中稳定出现 | ≥ 2 段 |
| 改变难度可评估 | 用户过去是否尝试改变过？成功率？ | 有尝试记录 |

### 提取流程

1. 列出 Phase 1-3 中所有发现的模式、矛盾、拐点
2. 按以上三个标准打分
3. 取 Top 2-4 个作为关键变量

**注意**: 大多数用户能提取 2-4 个关键变量，极少数可能只有 1 个或超过 5 个。如果只有 1 个 → 树只有一层但纵深更深。如果超过 5 个 → 取 Top 4。

---

## Step 2: 分支构建

每个关键变量生成：

```
[变量名称]
├─ 保持当前模式 (惯性分支)
│   └─ 标注: 用户过去在此变量上的行为一致性 + 改变难度
├─ 尝试改变 (改变分支)
│   └─ 标注: 改变方向基于 Phase 4 中对该变量的健康对照标准
└─ [可选] 极端化 (警示分支)
    └─ 仅在变量持续强化会导致显著负面后果时生成
```

---

## Step 3: 概率赋值

### 概率来源

- **P_user**: 用户自身行为数据 — 过去改变该行为的频率和成功率
- **P_pop**: 全国统计数据 — 同类人群的选择分布
- **P_research**: 心理学研究 — 特定人格/依恋风格的行为预测

### 综合概率公式

```
P(分支) = w1 × P_user + w2 × P_pop + w3 × P_research
其中 w1 + w2 + w3 = 1.0
默认权重: w1=0.4 (用户自身), w2=0.4 (人群数据), w3=0.2 (研究)
```

### 概率展示格式

```
达成概率: ~XX%
├─ 数据依据: [引用具体数据来源]
├─ 类似人群参考: [引用全国统计数据]
├─ 不确定因素: [列出无法量化的变量]
└─ ⚠️ 概率不是命运
```

---

## Step 4: 结局生成

### 六种结局类型

| 类型 | 定义 | 生成条件 |
|------|------|---------|
| 🍂 惯性 | 什么也不变 | 总是生成（基线对照） |
| 🌱 成长 | 关键改变 + 正向反应 | 变量中有"改变分支"且改变方向有数据支撑 |
| ⚡ 意外 | 外部随机事件 | 基于用户择偶模式生成可能的"相遇"/"变故" |
| 💀 警示 | 当前模式极端化 | 变量持续强化的负面后果有研究支撑 |
| 🔄 循环 | 表面改变实质重复 | 用户过去有"改变→回到原点"的记录 |
| 🌈 重构 | 核心观念根本转变 | 用户自述中表达过改变的强烈意愿 |

### 故事演绎规则

每个结局的故事必须包含:
1. 基于用户真实数据的具体细节（"你上次对XX说的那句话"而非"你会遇到一个人"）
2. 一个具体的场景（不是抽象描述）
3. 冲突或转折
4. 一个开放的结尾（"之后..."而非"从此幸福快乐"）

**禁止**:
- 编造用户没有提供的信息
- "从此幸福快乐"式的童话结尾
- 超过用户数据范围的过度推演

---

## 树的大小控制

- 关键变量: 2-4 个
- 每个变量的一级分支: 2-3 个（保持/改变/极端化）
- 随机事件层: 1-2 层
- 叶子结局: 6-15 个
- 每个结局故事: 3-5 句话

总计: 决策树 + 故事 = 适合一次对话上下文的大小
